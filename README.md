# Catalog Explorer
## 事前準備
- SQLウェアハウスの作成と権限の付与
  - アプリの「許可」タブの「アプリの認証」で、アプリで使用されているサービスプリンシパルのIDをコピー
  - 作成したSQLウェアハウスの「権限」から、サービスプリンシパルに対して「使用可能」権限を付与
- 確認したいデータに対する権限付与
  - 確認したいデータのカタログ、スキーマ、テーブルについて、サービスプリンシパルに対する「使用可能」権限を付与
- 使用するGenieスペースに対する権限付与
  - 確認したいGenieスペースについて、サービスプリンシパルに対する「実行可能」権限を付与

## 権限モデルについて：BROWSE 権限とデータアクセスの違い

このアプリでは、カタログやスキーマの一覧が権限を、明示的に設定していなくても表示される場合があります。
これは Unity Catalog の **BROWSE 権限** によるものです。

### BROWSE 権限とは

- Catalog Explorer（UI）で作成されたカタログには、デフォルトで **All account users** グループに `BROWSE` 権限が付与されます。アプリのサービスプリンシパルもこのグループに含まれます。
- `BROWSE` 権限があれば、`USE CATALOG` や `USE SCHEMA` が無くても、カタログ・スキーマ・テーブルの **名前やメタデータ（説明、タグ等）を閲覧** できます。
- 対象範囲：REST API、information_schema、リネージグラフ、検索結果など。

### BROWSE だけではデータは読めない

`BROWSE` はメタデータの閲覧のみを許可します。テーブルのデータ（SELECT）にアクセスするには、以下の権限がすべて必要です：

| 操作 | 必要な権限 |
| --- | --- |
| カタログ・スキーマ・テーブル一覧の表示 | `BROWSE`（デフォルトで付与される場合あり） |
| テーブルのデータ取得（SELECT） | `USE CATALOG` + `USE SCHEMA` + `SELECT` |
| テーブルへのデータ書き込み | `USE CATALOG` + `USE SCHEMA` + `MODIFY` |

### 注意事項

- **SQL 文（`CREATE CATALOG`）や REST API、Databricks CLI で作成されたカタログ** には、`BROWSE` がデフォルトで付与されません。必要に応じて以下を実行してください：
  ```sql
  GRANT BROWSE ON CATALOG <catalog_name> TO `account users`;
  ```

- カタログ一覧は見えるがサンプルデータが取得できない場合は、上記の `USE CATALOG` / `USE SCHEMA` / `SELECT` 権限が不足している可能性があります。

## 使い方

アプリにはサイドバー下部の **ページ選択** から切り替え可能な2つの機能があります。

---

### 📊 テーブル検索

Unity Catalog に登録されたテーブルのメタデータ・リネージ・サンプルデータを確認できる機能です。

#### 基本操作

1. **SQLウェアハウスを選択** — サイドバーの「SQLウェアハウスを選択」から、使用するウェアハウスを選びます。サンプルデータの取得に必要です。
2. **カタログ・スキーマを選択** — サイドバーのドロップダウンで、確認したいカタログとスキーマを絞り込みます（デフォルトは `samples.nyctaxi`）。
3. **テーブルを検索** — メインエリアの検索ボックスにテーブル名の一部を入力すると、テーブル一覧がフィルタリングされます。
4. **テーブル詳細を確認** — 一覧の上にあるドロップダウンからテーブルを選択すると、以下の情報が表示されます：

| 項目 | 内容 |
| --- | --- |
| 基本情報 | フルネーム、タイプ、データソース形式、ストレージパス |
| メタデータ | 作成者、更新者、コメント |
| カラム一覧 | カラム名、データ型、Nullable、コメント |
| テーブルLineage | 上流テーブル（Upstream）・下流テーブル（Downstream）の一覧 |
| サンプルデータ | スライダーで表示行数（10〜500行）を指定し、「サンプルデータを取得」ボタンで表示 |

### 🤖 Genie 問い合わせ

Databricks Genie を利用して、自然言語でデータに関する質問を行い、SQLの自動生成とクエリ結果の取得ができるチャット機能です。

#### 基本操作

1. **Genieスペースを選択** — サイドバーの「Genieスペースを選択」から、問い合わせ先のGenieスペースを選びます。スペースの説明がある場合はドロップダウンの下に表示されます。
2. **質問を入力** — 画面下部のチャット入力欄に、データに関する質問を自然言語で入力します（例：「先月の売上合計はいくらですか？」）。
3. **回答を確認** — Genieが回答を生成し、以下の内容が表示されます：
   * **テキスト回答** — Genieからの説明やコメント
   * **生成されたSQL** — 「生成されたSQL」を展開すると、Genieが自動生成したSQLクエリを確認できます
   * **クエリ結果** — SQLの実行結果がテーブル形式で表示されます
4. **会話を続ける** — 同じスペース内で連続して質問すると、会話のコンテキストが維持されます。文脈を踏まえた追加の質問が可能です。
5. **会話をリセット** — サイドバーの「会話をリセット」ボタンで、会話履歴をクリアして新しい会話を開始できます。
